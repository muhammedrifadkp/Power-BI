üìö Theory: Data Connection Fundamentals

Why Data Transformation Matters

Real-world data is messy! You'll often encounter:

  • Inconsistent formatting (UPPERCASE, lowercase, Mixed Case)
  • Extra spaces and special characters
  • Missing values
  • Wrong data types (numbers stored as text)
  • Unnecessary columns
  • Headers in wrong rows
  • Multiple data sources that need combining

Power Query Editor helps you clean and transform this data before it enters your data model.

The ETL Process

EXTRACT

Connect to data sources

Get data from files, databases, web, etc.

‚Üí

TRANSFORM

Clean and shape data

Remove errors, change types, filter, etc.

‚Üí

LOAD

Load into Power BI model

Ready for analysis and visualization

üìù Key Concept: Power Query uses the M language behind the scenes, but you don't need to write code - the interface generates it for you!

Data Quality Principles

Remember the 5 C's of Data Quality:

  1. Complete: No missing critical values
  2. Consistent: Same format throughout
  3. Clean: No errors or duplicates
  4. Correct: Accurate data types
  5. Current: Up-to-date information

Power Query helps you achieve all of these!

Connecting to Data: Step-by-Step Process

Step 1: Click Home ‚Üí Get Data

Applied Steps: The Power of Query Editor

What are Applied Steps?

Applied Steps is a history of all transformations applied to your data, in order.

Key Features:

  • üìù Each transformation creates a new step
  • üîÑ Steps are applied in sequence
  • üëÅÔ∏è Click any step to preview data at that point
  • ‚ùå Delete steps to undo transformations
  • ‚ÜïÔ∏è Reorder steps (with caution!)
  • ‚úèÔ∏è Rename steps for clarity
  • ‚öôÔ∏è Edit step settings by clicking the gear icon
‚ö†Ô∏è Important: The order of Applied Steps matters! Changing order can break your query or produce different results.

Common Applied Steps

Default Steps You'll See:

  1. Source: Connection to data source
  2. Navigation: Selected table/sheet
  3. Promoted Headers: First row as column names
  4. Changed Type: Auto-detected data types

Then you add your own transformations...

Data Profiling Tools

Enable in View tab:

  • Column Quality: Shows % valid, error, empty
  • Column Distribution: Shows distinct/unique counts
  • Column Profile: Shows value distribution histogram

üí° Pro Tip: Always enable these! They help identify data quality issues immediately.

üéØ Best Practice: Change profiling from "Top 1000 rows" to "Entire dataset" in the bottom status bar for accurate statistics.

üîÑ Basic Table Transformations

1. Working with Text Tools

Trim

Purpose: Remove leading/trailing spaces

Path: Transform ‚Üí Format ‚Üí Trim

Example:

" John " ‚Üí "John"

Clean

Purpose: Remove non-printable characters

Path: Transform ‚Üí Format ‚Üí Clean

Use: Fix text from external systems

Uppercase / Lowercase

Purpose: Change case

Path: Transform ‚Üí Format ‚Üí UPPERCASE

Options:

  • UPPERCASE
  • lowercase
  • Capitalize Each Word

Replace Values

Purpose: Find and replace text

Path: Transform ‚Üí Replace Values

Example:

Replace "USA" with "United States"

Split Column

Purpose: Divide one column into multiple

Path: Transform ‚Üí Split Column

Options:

  • By Delimiter
  • By Number of Characters
  • By Positions

Extract

Purpose: Pull out specific parts

Path: Transform ‚Üí Extract

Options:

  • First Characters
  • Last Characters
  • Range
  • Text Before/After Delimiter

Text Transformation Examples

üìù

Real-World Scenario: Cleaning Names

Problem: You have a name column with inconsistent formatting:

" JOHN SMITH "
"jane DOE"
"Bob Wilson"

Solution Steps:

  1. Select the column
  2. Transform ‚Üí Format ‚Üí Trim (removes extra spaces)
  3. Transform ‚Üí Format ‚Üí Capitalize Each Word

Result:

"Jane Doe"
"Bob Wilson"

3. Basic Column Operations

Remove Columns

Right-click column ‚Üí Remove

Or: Home ‚Üí Remove Columns

Tip: Use "Remove Other Columns" to keep only selected ones

Rename Columns

Double-click header

Or: Right-click ‚Üí Rename

Best Practice: Use clear, descriptive names

Change Data Type

Click type icon in header

Or: Transform ‚Üí Data Type

Types: Text, Number, Date, Boolean, etc.

Duplicate Column

Right-click ‚Üí Duplicate

Or: Add Column ‚Üí Duplicate Column

Use: Keep original while transforming

Move Columns

Drag column header

Or: Right-click ‚Üí Move

Options: To Beginning, To End, Left, Right

Filter Rows

Click filter icon in header

Select values to keep/remove

Types: Text, Number, Date filters

4. Handling Errors and Missing Data

Common Data Issues:

  • Null: No value exists
  • Error: Transformation failed
  • Blank: Empty string ""

Solutions:

  • Remove Errors: Home ‚Üí Remove Rows ‚Üí Remove Errors
  • Replace Errors: Transform ‚Üí Replace Errors
  • Fill Down: Transform ‚Üí Fill ‚Üí Down (copy value from above)
  • Fill Up: Transform ‚Üí Fill ‚Üí Up (copy value from below)
  • Replace Null: Transform ‚Üí Replace Values (leave "Value to Find" empty)
‚ö†Ô∏è Warning: Removing errors/nulls deletes entire rows! Consider replacing with default values instead.

Transformation Best Practices

‚úÖ DO's:
  • ‚úì Use descriptive names for queries and columns
  • ‚úì Document complex transformations
  • ‚úì Test transformations on sample data first
  • ‚úì Keep original columns when uncertain
  • ‚úì Use data profiling tools to check quality
  • ‚úì Apply transformations in logical order
‚ùå DON'Ts:
  • ‚úó Don't delete steps without understanding impact
  • ‚úó Don't over-complicate transformations
  • ‚úó Don't ignore errors or warnings
  • ‚úó Don't rely on column positions (use names)
  • ‚úó Don't transform in the data model (do it in Query Editor)

Exercise 2: Explore Query Editor Interface (10 minutes)

2

Familiarize Yourself

  1. Identify the main components:
    • Ribbon at top
    • Queries pane on left (shows "Sales" query)
    • Data preview in center
    • Query Settings on right (shows Applied Steps)
  2. Enable data profiling:

    • Check: Column Quality
    • Check: Column Distribution
    • Check: Column Profile
  3. In status bar (bottom), change from "based on top 1000 rows" to "based on entire data set"
  4. Observe the Applied Steps on the right

Exercise 3: Clean Product Names (15 minutes)

3

Text Transformations

Scenario: Product names have extra spaces and inconsistent capitalization

  1. Select the Product column (click header)
  2. Go to Transform ‚Üí Format ‚Üí Trim
    • Notice a new step appears in Applied Steps
  3. With column still selected, Transform ‚Üí Format ‚Üí Capitalize Each Word
  4. Review your data - product names should look clean now!
  5. In Applied Steps, rename:
    • "Trimmed Text" ‚Üí "Trim Product"
    • "Capitalized Each Word" ‚Üí "Capitalize Product"

Exercise 4: Fix Region Column (15 minutes)

4

Replace Values

Scenario: Region column has abbreviations that need to be expanded

  1. Select the Region column
  2. Go to Transform ‚Üí Replace Values
  3. Replace:
    • Value to Find: N
    • Replace With: North
    • Click OK
  4. Repeat for:
    • S ‚Üí South
    • E ‚Üí East
    • W ‚Üí West
  5. Verify all regions are now full names

Exercise 5: Work with Numbers (15 minutes)

5

Numerical Transformations

Task A: Round Sales Values

  1. Select the Sales column
  2. Check the data type icon - should be decimal/number
  3. If it's text, change it: Click type icon ‚Üí Decimal Number
  4. Go to Transform ‚Üí Rounding ‚Üí Round
  5. Enter 2 for 2 decimal places
  6. Click OK

Task B: Create a Calculated Column

  1. Go to Add Column tab
  2. Click Custom Column
  3. New column name: Total Value
  4. Formula: = [Sales] * [Quantity]
  5. Click OK
  6. You now have a new column calculating total value!

Exercise 6: Handle Missing Data (10 minutes)

6

Deal with Nulls

  1. Look at the Column Quality indicators
    • Notice if any column shows "Empty" percentage
  2. Select a column with null values (if any)
  3. Right-click column header ‚Üí Replace Values
  4. Leave "Value to Find" empty (this finds nulls)
  5. Replace With: Unknown or 0 (depending on column)
  6. Click OK
  7. Alternative: Home ‚Üí Remove Rows ‚Üí Remove Blank Rows

Exercise 7: Filter and Remove Columns (10 minutes)

7

Final Cleanup

Task A: Filter Data

  1. Click filter icon on Quantity column
  2. Choose Number Filters ‚Üí Greater Than
  3. Enter 0
  4. Click OK (removes rows with 0 or negative quantity)

Task B: Remove Unnecessary Columns

  1. Hold Ctrl and click to select multiple columns you don't need
  2. Right-click ‚Üí Remove Columns
  3. Or use Home ‚Üí Remove Columns ‚Üí Remove Other Columns to keep only selected ones

Exercise 8: Load Transformed Data (5 minutes)

8

Apply Changes

  1. Review all your Applied Steps on the right
  2. Click through steps to see data at each stage
  3. Rename the query:
    • In Query Settings, change "Sales" to Sales_Clean
  4. Click Home ‚Üí Close & Apply
  5. Wait for data to load into Power BI Desktop
  6. You're back in Report View!

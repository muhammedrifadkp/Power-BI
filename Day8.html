üìä Choosing the Right Visual Type

The Visual Selection Framework

Ask yourself these questions:

  1. What am I trying to show? (Comparison, Trend, Part-to-Whole, Distribution, Relationship)
  2. How many variables? (One, Two, Multiple)
  3. What's the data structure? (Time series, Categories, Geographic, Hierarchical)
  4. Who's the audience? (Executive, Analyst, Operational)

Visual Types by Purpose

üìà Comparison Visuals

Bar Chart (Clustered/Stacked)
üìä

Best for: Comparing values across categories

When to use:

  • Compare sales by product
  • Compare regions or segments
  • Horizontal bars for long category names

Max categories: 10-15 for readability

Column Chart
üìä

Best for: Comparing values over time periods

When to use:

  • Monthly/Quarterly comparisons
  • Year-over-year analysis
  • Vertical orientation preferred
Table & Matrix
üìã

Best for: Detailed data lookup

When to use:

  • Display exact values
  • Multi-dimensional analysis
  • Drill-down hierarchies (Matrix)

üìâ Trend Visuals

Line Chart
üìà

Best for: Showing trends over time

When to use:

  • Continuous time series
  • Multiple trend lines comparison
  • Spotting patterns and anomalies

Add: Trend lines, forecasting

Area Chart
üìä

Best for: Showing cumulative totals over time

When to use:

  • Emphasize magnitude of change
  • Stacked to show part-to-whole
  • Volume over time
Combo Chart
üìä

Best for: Comparing different scales

When to use:

  • Revenue (bars) + Profit % (line)
  • Volume + Price trends
  • Different measurement units

ü•ß Part-to-Whole Visuals

Pie Chart
ü•ß

Best for: Simple proportions (2-5 slices)

When to use:

  • Market share breakdown
  • Budget allocation
  • Limited categories only!

Donut Chart
üç©

Best for: Same as Pie, with center space

When to use:

  • Display total in center
  • Slightly more modern look
  • 2-5 categories
Treemap
üó∫Ô∏è

When to use:

  • Many categories (6-20)
  • Nested hierarchies
  • Product portfolio analysis
100% Stacked Bar/Column
üìä

Best for: Comparing proportions across groups

When to use:

  • Category composition over time
  • Relative percentages
  • Distribution comparison

  • Progress indicators

Visuals to Avoid (Usually)

‚ö†Ô∏è Use Sparingly:

  • 3D Charts: Distort perception, hard to read exact values
  • Pie with many slices: Hard to compare, use bar chart instead
  • Dual-axis with misleading scales: Can be manipulative
  • Radar/Spider charts: Difficult to interpret accurately

üé® Report Formatting & Styling

Accessing Format Options

Three ways to format:

  1. Visual Format Pane: Select visual ‚Üí Format icon (paint roller) ‚Üí Customize
  2. Report Format Pane: Click canvas ‚Üí Format icon ‚Üí Page settings
  3. Themes: View ‚Üí Themes ‚Üí Choose or customize

Visual Formatting Options

General Settings

  • Title: On/Off, Text, Font, Color, Background
  • Effects: Background color, Border, Shadow, Glow
  • Tooltips: Default or Report page
  • Position: X, Y, Width, Height

Data Colors

  • Default colors: Single color for all
  • Show all: Different color per category
  • Conditional formatting: Color by value
  • Custom colors: Brand colors

Data Labels

  • Toggle: Show/Hide values on chart
  • Position: Inside, Outside, Auto
  • Format: Font, Size, Color
  • Units: Display units (K, M, B)

Axis Settings

  • Range: Auto or Manual min/max
  • Scale: Linear or Logarithmic
  • Labels: Font, Color, Rotation
  • Gridlines: Show/Hide, Style

Legend

  • Position: Top, Bottom, Left, Right
  • Title: Custom text
  • Font: Size and Color
  • Show/Hide: Toggle visibility

Background & Border

  • Color: Solid or Gradient
  • Transparency: 0-100%
  • Border: Color, Width, Radius
  • Effects: Shadow, Glow

Conditional Formatting

Where to Apply:

  • Table/Matrix: Background colors, Font colors, Data bars, Icons
  • Cards: Background based on value
  • Charts: Data colors based on value

How to Apply:

  1. Select visual
  2. Format pane ‚Üí Find relevant section (e.g., Cell elements for Matrix)
  3. Click fx icon next to color option
  4. Choose:
    • Rules: If value > X then color Y
    • Gradient: Low to High color scale
    • Field value: Color based on another column

Example: Traffic Light Colors

-- For Profit Margin Red: Values < 10% Yellow: Values 10-20% Green: Values > 20%

Report Themes

Themes: Pre-defined color schemes and fonts applied to entire report

How to use:

  1. View tab ‚Üí Themes ‚Üí Choose built-in theme
  2. Or customize: View ‚Üí Themes ‚Üí Customize current theme
  3. Or import: Browse for theme ‚Üí Upload JSON file

What themes control:

  • Primary colors (8-10 colors)
  • Font family and sizes
  • Background colors
  • Visual defaults

Design Best Practices

‚úÖ DO's

  • Use consistent colors (brand colors)
  • Limit to 3-4 main colors
  • Align visuals to grid
  • Use white space effectively
  • Make titles descriptive
  • Keep font sizes readable (min 10pt)
  • Remove chart junk (unnecessary elements)
  • Test on different screen sizes

‚ùå DON'Ts

  • Don't use too many colors
  • Don't make visuals too small
  • Don't use 3D effects
  • Don't overcrowd pages
  • Don't use clashing colors
  • Don't forget accessibility (color blind)
  • Don't ignore alignment
  • Don't use tiny fonts

Page Formatting

üìÑ

Page-Level Settings

Access: Click canvas ‚Üí Format pane

  • Canvas background: Color, Image, Transparency
  • Page size: 16:9, 4:3, Letter, Custom
  • Wallpaper: Background image
  • Page information: Name, Description

üîó Visual Interactions & Filtering

Understanding Visual Interactions

By default: When you click on a visual, it filters or highlights other visuals on the same page

Three interaction types:

  • Filter: Selected visual filters the target visual (most common)
  • Highlight: Shows selected portion highlighted, dimming the rest
  • None: No interaction occurs

Editing Interactions

üéõÔ∏è

How to Edit Interactions

  1. Select the SOURCE visual (the one that will do the filtering)
  2. Go to Format tab ‚Üí Edit interactions
  3. Icons appear above each target visual:
    • üîç Filter icon
    • üéØ Highlight icon
    • ‚äò None icon
  4. Click desired interaction type for each target
  5. Click Edit interactions again to finish

Common Scenario: KPI Cards

You usually DON'T want other visuals to filter your KPI cards showing totals

Solution:

  1. Select slicer or chart
  2. Edit interactions
  3. Set KPI cards to "None"

Filter Types in Power BI

Visual-level Filters

Scope: Single visual only

Use: Limit data shown in one visual

Example: Show only Top 10 products

Page-level Filters

Scope: All visuals on current page

Use: Filter entire page context

Example: Show only Current Year data

Report-level Filters

Scope: All pages in report

Use: Global filters

Example: Exclude test data

Slicers

Scope: Visible to users

Use: Interactive filtering

Example: Date range, Category selection

Drill-Through Pages

Drill-through: Navigate from summary to detailed page, maintaining context

How it works:

  1. Create a detail page (e.g., "Product Details")
  2. Add field(s) to Drill-through wells (e.g., ProductName)
  3. Add detailed visuals
  4. On summary page: Right-click data point ‚Üí Drill through ‚Üí Product Details
  5. Detail page opens with context filtered
üîç

Creating Drill-Through

  1. Create new page: "Product Detail"
  2. In Filters pane ‚Üí Drill through section
  3. Drag ProductName to drill-through field
  4. Add visuals showing product details
  5. Power BI automatically adds back button
  6. Test: Go to summary ‚Üí Right-click product ‚Üí Drill through

Cross-Filtering vs Cross-Highlighting

Aspect Cross-Filtering Cross-Highlighting
Behavior Hides non-matching data Dims non-matching data
Use When Want to see only relevant data Want to compare selected vs total
Visual Impact Data disappears Data stays but grayed out
Best For Detailed analysis Seeing proportions

‚ö° Advanced Report Features

Bookmarks for Navigation

Bookmarks: Capture the current state of a report page (filters, visual visibility, selection)

Use cases:

  • Create navigation between pages
  • Show/hide visuals dynamically
  • Tell a data story
  • Create multiple views of same page
üîñ

Creating Bookmarks

  1. View tab ‚Üí Bookmarks Pane
  2. Set up your page (filters, selections, visible visuals)
  3. Click Add in Bookmarks pane
  4. Rename bookmark (e.g., "Sales View")
  5. Create more bookmarks with different states
  6. Test: Click bookmark names to switch views

Advanced: Link to Buttons

  1. Insert ‚Üí Button ‚Üí Blank or icon
  2. Select button ‚Üí Format ‚Üí Action ‚Üí On
  3. Type: Bookmark
  4. Bookmark: Choose your bookmark
  5. Test in Reading View

Selection Pane

Selection Pane: Manage visual visibility and layering

Access: View ‚Üí Selection Pane

Features:

  • Show/Hide: Eye icon to toggle visibility
  • Layer order: Drag to reorder (top = front)
  • Group: Group related visuals
  • Rename: Give visuals meaningful names

Tooltips (Custom Report Pages)

Tooltip pages: Show additional context on hover without leaving main page

How to create:

  1. Create new page
  2. Format ‚Üí Page information ‚Üí Type: Tooltip
  3. Set page size (Canvas settings ‚Üí Type: Tooltip)
  4. Add visuals (they'll show on hover)
  5. On main page: Select visual ‚Üí Format ‚Üí Tooltips ‚Üí Report page
  6. Choose your tooltip page

Mobile Layout

üì±

Optimizing for Mobile

  1. View tab ‚Üí Mobile layout
  2. Canvas changes to phone dimensions
  3. Drag visuals from right pane to phone canvas
  4. Resize and reposition for mobile
  5. Not all visuals need to be included
  6. Prioritize most important KPIs

üí° Tip: Design mobile-first for executive dashboards

Buttons & Navigation

Button Types

  • Blank: Custom text/icon
  • Back: Return to previous page
  • Bookmark: Navigate to bookmark
  • Drill through: Go to detail page
  • Page navigation: Go to specific page
  • Q&A: Open Q&A interface
  • Web URL: External link

Button States

  • Default: Normal appearance
  • On hover: Mouse over effect
  • On press: Click effect
  • Each state can have different formatting

Custom Visuals from AppSource

AppSource: Marketplace for additional visuals beyond built-in ones

How to add:

  1. Visualizations pane ‚Üí "..." ‚Üí Get more visuals
  2. Browse or search AppSource
  3. Click visual ‚Üí Add
  4. Visual appears in Visualizations pane

Popular custom visuals:

  • Chiclet Slicer: Better slicer with images
  • Text Filter: Search box filter
  • Bullet Chart: Compare actual to target
  • Timeline Slicer: Date range with slider
  • Word Cloud: Text frequency visualization
  • Sankey Diagram: Flow visualization
‚ö†Ô∏è Custom Visual Cautions:
  • Not all are created by Microsoft (review publisher)
  • May have performance implications
  • Some require special licensing
  • Test thoroughly before deploying

üî® Practical Lab: Building a Professional Dashboard

‚è±Ô∏è Lab Duration: 90 minutes
üéØ Objective: Create a multi-page professional sales dashboard with advanced features

Exercise 1: Executive Summary Page (30 minutes)

1

Create KPI Dashboard

  1. Create new page: "Executive Summary"
  2. Apply theme: View ‚Üí Themes ‚Üí Choose one
  3. Add KPI Cards (top row):
    • YTD Sales
    • YoY Growth %
    • Total Customers
    • Avg Order Value
  4. Format cards:
    • Large font (32pt+)
    • Background color
    • Border and shadow
  5. Add Line Chart (sales trend over time)
  6. Add Treemap (sales by category)
  7. Add Map (sales by region)
  8. Add slicers (Year, Quarter)
  9. Align all visuals to grid

Exercise 2: Product Analysis Page (20 minutes)

2

Detailed Product View

  1. Create new page: "Product Analysis"
  2. Add Matrix visual:
    • Rows: Products[Category], Products[ProductName]
    • Values: [Total Sales], [% of Total], [YoY Growth %]
    • Enable drill-down on Category
  3. Apply conditional formatting:
    • Data bars on Total Sales
    • Color scale on YoY Growth % (red-yellow-green)
  4. Add Bar Chart (Top 10 products by revenue)
  5. Add Column Chart (Product sales over time)

Exercise 3: Configure Interactions (15 minutes)

3

Set Up Filtering Behavior

  1. Go to Executive Summary page
  2. Select Year slicer
  3. Format ‚Üí Edit interactions
  4. Set KPI cards to "None" (don't filter totals)
  5. Keep other visuals as "Filter"
  6. Select Treemap
  7. Edit interactions ‚Üí Set Map to "Highlight"
  8. Test interactions

Exercise 4: Create Drill-Through (15 minutes)

4

Product Detail Page

  1. Create new page: "Product Detail"
  2. Filters pane ‚Üí Drill through ‚Üí Add Products[ProductName]
  3. Add visuals:
    • Card showing product name
    • Cards for metrics (Total Sales, Units Sold, Avg Price)
    • Line chart (sales trend for this product)
    • Table (customer list who bought this product)
  4. Test: Go to Product Analysis ‚Üí Right-click product ‚Üí Drill through ‚Üí Product Detail

Exercise 5: Add Bookmarks (10 minutes)

5

Create View States

  1. View ‚Üí Bookmarks pane
  2. On Executive Summary, create bookmark "Default View"
  3. Select Year = 2024, create bookmark "2024 View"
  4. Select Year = 2023, create bookmark "2023 View"
  5. Insert buttons:
    • Add three buttons labeled "All Years", "2024", "2023"
    • Link each to corresponding bookmark
  6. Test in Reading View

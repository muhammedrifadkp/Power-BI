üí¨ Q&A Visual - Natural Language Queries

What is Q&A Visual?

Q&A Visual: Allows users to ask questions about data in natural language and get visual answers automatically.

How it works:

  1. User types a question (e.g., "total sales by region")
  2. Power BI interprets the question
  3. Creates appropriate visualization
  4. User can refine or ask follow-up questions

Benefits:

  • Enables self-service analytics for non-technical users
  • Reduces need for pre-built visuals
  • Accelerates data exploration
  • Natural, conversational interface

Adding Q&A Visual

1

How to Add Q&A

  1. Go to Report View
  2. Insert tab ‚Üí Q&A button
  3. Q&A visual appears with question box
  4. Resize to fit your layout
  5. Users can now type questions

Example Questions

Sample Questions Users Can Ask:

"total sales by category"

‚Üí Creates bar chart

"show revenue over time"

‚Üí Creates line chart

"top 5 products by revenue"

‚Üí Creates sorted bar chart

"average order value"

‚Üí Creates card with number

"sales by region as map"

‚Üí Creates filled map

"what drove sales increase"

‚Üí Analyzes key factors

Teaching Q&A to Understand Your Data

Q&A Setup (Power BI Service Only):

  1. Publish report to Power BI Service
  2. Open in service ‚Üí Q&A Setup
  3. Add synonyms (e.g., "revenue" = "sales")
  4. Define suggested questions
  5. Review and train on user questions

Best Practices:

  • Use clear, descriptive field names
  • Add synonyms for common terms
  • Create suggested questions for guidance
  • Test with actual user questions

Converting Q&A to Standard Visual

üí°

Save Q&A Results

If a user asks a good question:

  1. After Q&A creates visualization
  2. Click "Turn this Q&A result into a standard visual"
  3. Visual becomes permanent on the page
  4. Can be formatted like any other visual
üí° Pro Tip: Place Q&A visual on a dedicated "Explore" page for ad-hoc analysis!

üéØ Key Influencers Visual

What is Key Influencers?

Key Influencers: AI-powered visual that analyzes what factors influence a metric to increase or decrease.

Use cases:

  • What drives customer churn?
  • What influences high sales?
  • What factors affect satisfaction scores?
  • What drives product returns?

Two tabs:

  • Key Influencers: Shows top factors influencing the metric
  • Top Segments: Shows groups with highest/lowest values

Setting Up Key Influencers

2

How to Configure

  1. Insert ‚Üí AI Visuals ‚Üí Key Influencers
  2. Configure fields:
    • Analyze: Metric to analyze (e.g., Sales, CustomerRating)
    • Explain by: Potential influencing factors (Category, Region, Product, etc.)
    • Expand by: (Optional) Additional drill-down fields
  3. Choose what to analyze: "Increase" or "Decrease"
  4. Power BI analyzes and shows results

Example: What Influences High Sales?

Scenario: Analyze what drives sales above $1000

Setup:

  • Analyze: Sales[Revenue]
  • Explain by: Products[Category], Customers[Region], Date[Year]
  • Filter: Revenue > 1000

Possible Results:

  • "Sales is 2.5x more likely to be high when Category is Electronics"
  • "Sales is 1.8x more likely to be high when Region is North"
  • "Sales increases by avg $450 when Product is Laptop Pro"

Reading Key Influencers Results

Understanding the Output:

For Categorical Metrics (e.g., High/Low Rating):

  • "X is Y times more likely when..."
  • Shows probability multiplier
  • Bubble size = number of records

For Continuous Metrics (e.g., Sales Amount):

  • "On average, X increases/decreases by..."
  • Shows average impact
  • Bar length = strength of influence

Top Segments Tab:

  • Shows groups with highest concentration
  • Example: "Electronics + North Region = 85% high sales"
  • Helps identify target segments

Best Practices

‚úÖ DO's

  • Use with sufficient data (100+ rows minimum)
  • Include diverse potential influencers
  • Analyze meaningful metrics
  • Consider correlation vs causation

‚ùå DON'Ts

  • Don't use with very small datasets
  • Don't assume causation from correlation
  • Don't ignore domain knowledge
  • Don't over-interpret results
‚ö†Ô∏è Remember: Key Influencers shows correlation, not necessarily causation. Always apply business knowledge when interpreting results!

üå≥ Decomposition Tree Visual

What is Decomposition Tree?

Decomposition Tree: AI-powered visual for ad-hoc hierarchical exploration of data.

Key features:

  • Start with a metric (e.g., Total Sales)
  • Drill down by any dimension
  • AI suggests best splits
  • Interactive exploration
  • Find high/low contributors quickly

Use cases:

  • Root cause analysis
  • Finding outliers
  • Exploratory data analysis
  • Multi-dimensional drill-down

Setting Up Decomposition Tree

3

Configuration Steps

  1. Insert ‚Üí AI Visuals ‚Üí Decomposition Tree
  2. Configure fields:
    • Analyze: The metric (e.g., Sum of Sales)
    • Explain by: All dimensions you might drill by (Product, Region, Date, Customer, etc.)
  3. Visual starts with total metric value
  4. Click "+" to drill down

Using the Decomposition Tree

Interactive Exploration:

Level 0: Total Sales: $1,250,000

‚Üì Click "+" ‚Üí Choose dimension

Level 1: Split by Category

  • Electronics: $650,000 (52%)
  • Furniture: $400,000 (32%)
  • Stationery: $200,000 (16%)

‚Üì Click "+" on Electronics

Level 2: Split by Region

  • North: $280,000
  • South: $220,000
  • East: $150,000

AI-Powered Features

High Value & Low Value:

When you click "+", you get three options:

  • Manual split: Choose dimension yourself
  • High value: AI finds path to highest values
  • Low value: AI finds path to lowest values

Example:

Click "High value" on Total Sales:

  • AI might split by "Category = Electronics"
  • Then "Region = North"
  • Then "Product = Laptop Pro"
  • Shows path to highest sales

Decomposition Tree vs Matrix

Feature Decomposition Tree Matrix
Structure Flexible, user-driven Fixed hierarchy
Exploration Ad-hoc, any path Pre-defined drill-down
AI Assistance Yes (High/Low value) No
Best For Root cause analysis, exploration Systematic reporting

üåê Publishing to Power BI Service

Power BI Desktop vs Power BI Service

Feature Power BI Desktop Power BI Service
Platform Windows application Cloud web service
Purpose Create & develop reports Share & collaborate
Data Modeling Full capabilities Limited
Report Design Full capabilities View & light editing
Sharing Share .pbix file Share links, embed
Data Refresh Manual Scheduled automatic
Collaboration Limited Full (workspaces, apps)

Publishing Process

üì§

How to Publish

  1. In Power BI Desktop, ensure you're signed in
    • Top right ‚Üí Sign in with work/school account
  2. Save your .pbix file
  3. Click Home ‚Üí Publish
  4. Choose destination workspace:
    • My Workspace (personal)
    • Shared workspace (team)
  5. Click Select
  6. Wait for publishing to complete
  7. Click link to open in Power BI Service

Power BI Service Overview

URL: app.powerbi.com

Main Components:

  • Workspaces: Containers for reports, dashboards, datasets
  • Reports: Interactive multi-page reports
  • Dashboards: Single-page pinned visuals from reports
  • Datasets: Data models
  • Dataflows: Reusable data prep
  • Apps: Published collections for distribution

Sharing Reports

Method 1: Share Link

How: Report ‚Üí Share ‚Üí Copy link

Who: Anyone with link (if allowed)

Permissions: Read-only or can reshare

Best for: Quick sharing

Method 2: Email

How: File ‚Üí Share ‚Üí Email

Who: Specific people

Features: Can include snapshot

Best for: Scheduled distributions

Method 3: Publish to Web

How: File ‚Üí Embed ‚Üí Publish to web

Who: Public, anyone on internet

Warning: Data becomes public!

Best for: Public dashboards

Method 4: App

How: Create app from workspace

Who: Specific users/groups

Features: Professional distribution

Best for: Enterprise deployment

Data Refresh

Setting Up Scheduled Refresh:

  1. In Power BI Service, go to workspace
  2. Find dataset ‚Üí More options (...) ‚Üí Settings
  3. Expand Scheduled refresh
  4. Configure:
    • Refresh frequency (Daily, Weekly)
    • Time slots
    • Time zone
    • Email notifications on failure
  5. Apply

Requirements:

  • Data gateway for on-premises data
  • Pro or Premium license
  • Valid credentials configured
‚ö†Ô∏è Licensing Note: Free license allows publishing but limited sharing. Pro/Premium required for full collaboration features.
